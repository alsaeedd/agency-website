# RAL — Design Context

This file is the source of truth for design intent on the RAL (Revenue Automation Lab) site. It is loaded by Claude Code (and humans) before any design or frontend work, so design decisions stay coherent across sessions. Update this file whenever the direction shifts.

## Design Context

### Users

The site serves **two co-equal audiences**, neither of which can be alienated:

1. **GCC founders & SMB owners** — non-technical decision makers vetting RAL's credibility in 30 seconds. They want warmth, a clear sense of "is this for me?", and a low-friction way to start a conversation. They will not read long case studies. They will read tone.
2. **Enterprise / corporate procurement & tech leads** — vetting RAL as a vendor against larger consultancies. They want case-study depth, technical proof, and signal that RAL can deliver at scale.

**The job-to-be-done across both groups**: form a fast trust judgment, then either start a conversation (small clients) or drill into a project case study (larger clients). The site is a vetting tool first, a sales tool second.

When designing, default to the lowest common denominator (warmth + clarity) and offer **optional depth** for the procurement audience (project detail overlays, technical specifics in case studies). Never make the depth a barrier to the warmth.

### Brand Personality

**Three words**: confident · founder-led · technically serious.

**Voice**: first-person, warm, conversational. Real examples already in the codebase that capture it well:
- "We build your solution as if we literally owned it."
- "So, what're we cooking?"
- "Tell us what you're working on and we'll get back to you shortly."

Avoid corporate passive voice. Avoid hype words ("revolutionize", "disrupt", "next-generation"). Avoid adjective stacks. Address the visitor like a peer — not a prospect, not a stakeholder.

**Emotional goals**:
- A Citibank procurement lead trusts that RAL can ship.
- A Bahraini founder with a 1k-BHD budget feels welcomed, not priced out.
- Both feel the site was built by humans who care, not assembled from a template.

### Aesthetic Direction

**Theme**: dark only. No light-mode toggle. The dark-purple atmosphere is core to the brand.

**Visual language** (locked in, do not drift from):
- Background `#0c0715` (near-black with violet cast), surfaces `#160e27` and `#2b1b3d`
- Primary text `#f8f8f8`, secondary `#d1c4e9` (lavender)
- Accent `#a78bfa` (lavender purple) for active states, links, focus rings
- Pulse `#4af5c0` (mint) reserved for "live" / status indicators only
- Translucent borders (`rgba(209,196,233,0.1)` baseline, `0.22` active)
- Inter font, weights 300–700, fluid `clamp()` sizing, tight letter-spacing on display (-0.02 to -0.04em), loose on eyebrows (0.1 to 0.28em)
- Spline 3D brain as hero centerpiece (desktop + mobile scenes)
- Lava-lamp blurred radial blobs as fixed background atmosphere
- Glass nav with `backdrop-filter: blur(20px)`
- GSAP scroll-pinned timelines for major sections (Hero, About, Services, CTA)
- Word-by-word and letter-by-letter reveals via `AnimatedText` component
- Blur-in entrance animations (filter-blur 6–24px → 0)

**Tension to preserve**: futuristic visuals + warm voice. The two layers must coexist. The Spline brain over "Build Your Dream Project" is the canonical example. When the visual layer wins too hard the brand reads cold; when the voice layer wins too hard it reads small.

**Direction going forward**: lean **more warm / founder-led, less elite / futuristic**. The visual density is already at its ceiling. New work should add personality (copy, micro-interactions, human texture, cultural touches like the gutra-guy contact circle) rather than more polish (more glows, more gradients, more scroll-pins).

**Anti-references** — kill anything that drifts toward these:
- Generic dev-shop / template agency (three-icon-grid, blue accent, stock photo of laptop, bootstrap-y feel)
- AI-slop landing page (gradient-text-everywhere, stacks of glass cards, default shadcn look, emoji-heavy)
- Corporate / consulting firm (Big Four navy + Helvetica + stock-photo-of-handshake sterility)
- Crypto / web3 hype (neon green, cyberpunk fonts, "we're disrupting X", NFT-grid vibes)

**Accessibility & reduced motion**: currently **not implemented and explicitly deprioritized**. No `prefers-reduced-motion` handling, no formal WCAG audit. This is a known and accepted gap. Revisit when reaching for international/enterprise contracts that mandate it. Do not water down motion to retroactively address this — instead, add a clean reduced-motion fallback path when the time comes.

### Design Principles

1. **Warmth wins ties.** When pulled between premium-elite polish and founder-led warmth, lean warm. Visual density is at ceiling — new work should add personality (copy, micro-interactions, human texture) not more polish (glows, gradients, scroll-pins).

2. **Two audiences, no compromise.** Every page must read as credible to enterprise procurement AND welcoming to a founder with a small budget. Test against both before shipping. If either feels alienated, the design fails.

3. **Specificity beats impressiveness.** "1,000 candidates messaged per batch" beats "enterprise-scale outreach automation." Numbers, real client names, real outcomes, concrete artifacts — always win over adjectives.

4. **Kill the four anti-aesthetics.** If a design choice drifts toward (a) generic dev-shop template, (b) AI-slop landing page, (c) corporate-consulting sterility, or (d) crypto/web3 hype — rework it. The dark-purple + Spline + warm-copy language is the antidote. Don't drift back.
